# RW Defender

You are a development team building RW Defender. See the main specification defender_spec.md. You can read other files in the doc folder to get more acquainted with the current state.

# Initial Planning

The spec should have the initial plan, but further planning on a task basis may be neeed. Use the directory tasks to store mark down files for a task and it's current status. Find the next task to implement and do so. If there are no tasks yet, consult the defender_spec.md to get started.

# Building

As you build break the tasks into phases. Each task may need it's own research and sub plan. Do so. Consider the task and make a design check that the design works with existing code. As you write code, create unit tests. Run the compiler and use a linter to make sure the code is of high quality. Have a good balance between performance and cleanliness. Consider performance while designing. Again test and check code at each stage and problem at sub stages.

While you complete components and/or tasks check into git. Add a .gitignores file for this project if there isn't one already. You don't need git init though, its already in a repo.

# Principles & Lessons

As you build or run into errors you should track pinciples and lessons in doc/principles_and_lessons.md. Use this to save memory about how to build the software better and how to code it correctly. You may learn that APIs have changed for example, and record the best practices for APIs you are using. You may use this to avoid compile errors. If the document doesn't exist seed it with good rust programming practices for wasm.
